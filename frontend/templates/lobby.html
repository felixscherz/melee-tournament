HOW TO WRITE A BOT

There are three ways to control a player:

  • Default AI — leave everything blank. A built-in bot plays the character.
  • Custom code — paste a Python Bot class into the code box.
  • Generate from prompt — describe how the bot should play in the prompt box and click GENERATE. An AI agent writes and tests the code for you. This takes 30s-5min.

If both custom code and a generated bot exist, the custom code takes priority.

Your act(gamestate, player_port) must return a dict shaped like:

{
  "stick_x": 0.5,   # 0.0 = left, 1.0 = right, 0.5 = neutral
  "stick_y": 0.5,   # 0.0 = down, 1.0 = up,    0.5 = neutral
  "buttons": {
    "BUTTON_A": False, "BUTTON_B": False,
    "BUTTON_X": False, "BUTTON_Y": False,
    "BUTTON_L": False, "BUTTON_R": False,
    "BUTTON_Z": False,
  },
}

Useful gamestate fields: gamestate.players[port].position.x / .y, .stock, .percent, .action, .character.

SAFETY RULES (enforced on submit)

  • Only import melee (plus math, random) is allowed — any other import is rejected.
  • eval, exec, compile, open, __import__ and similar are banned.
  • Dunder tricks like ().__class__.__bases__ are rejected.
  • A top-level Bot class with an act method is required.
